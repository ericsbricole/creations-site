URGI (Unité de Recherche Génomique Info) - INRA, 03/2017 - 08/2017(6 mois), Versailles, stage de M2

PASTEClassifier (plus simplement PASTEC, pour Pseudo Agent System for Transposable Element Classification) est un logiciel de classification automatique des éléments transposables d'un génome développé à l'URGI avec le language python. Il effectue sa classification à partir de règles de décisions définies dans des fichiers au format yaml. Mon travail a consisté à développer un outil python en ligne de commande, prenant en entrée les fichiers de résultats de PASTEC obtenus sur deux configurations différentes, afin d' afficher une comparaison claire des différences de résultats à l'aide de fichiers texte et de graphiques. Cet outil doit permettre à l'équipe d'analyse des génomes de l'URGI de choisir quelle configuration est la meilleure pour l'analyse d'un échantillon d'ADN donné.

eventPositionner


Il s'agit d'une mappemonde 2D interactive permettant de rentrer un fichier de données au format csv et de visualiser sur la carte les pays où les valeurs sont les plus importantes, à l'aide d'une légende automatique séparant les pays en 4 groupes égaux. Ce travail personnel est inspiré d'un projet de développement validant la fin de mon M1. Je l'ai développé en JavaScript avec les bibliothèques d3js, jQuery et jQueryUI. La partie html bénéficie quant à elle du framework css Bootstrap, ce qui lui permet de s'adapter à toutes les tailles d'écran («responsive design»). Les formes des pays proviennent d'un fichier shapefile (.sh) du site NaturalEarth. J'ai utilisé le logiciel QGis pour convertir ces données en GeoJson. Vous pouvez tester ce programme car il tourne, comme ce site, sur mes pages github.



Déminons


Déminons est une version paramétrable du célèbre jeu de démineur, développé en Java 8 avec la bibliothèque graphique JavaFX. Le joueur peut choisir la largeur et la hauteur de son plateau, ainsi que le nombre de bombes. Ce projet est nommé ainsi car je souhaitais en faire une application jouable à deux avec un espace de dialogue (chat). Cependant, j'ai appris que cela obligerait les joueurs à faire de la redirection de ports. Ne souhaitant pas obliger mes utilisateurs à faire une manipulation qui puisse les rendre mal à l'aise, j'ai décidé de ne pas implémenter cette fonctionnalité. Je n'exclus pas de revenir un jour sur cette décision, mais j'ai préféré développer en priorité d'autres projets dont les fonctionnalités seraient explorables sans manipulation de la part de l'utilisateur.
